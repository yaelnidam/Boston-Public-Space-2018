Issues Faced & Potential Alternative Approaches

  • Initially, when analyzing the flood models, we did not have a raster set that we could use to highlight how many feet certain areas would be flooded. Until we were able to receive this dataset, our model only said "Flooded" or "Not Flooded".
  • Figuring out the best way to show the data took a lot of work. Admittedly, some next steps in the project is to add the layers to the individual park maps.
  • There are multiple scenarios for sea level rising and it was hard to determine which one to use.

Data Processing Procedures for Interactive Maps

  1. Google: Draw routes on Google Maps and export to .KML
  2. R: Scrape coordinates into csv
  3. R: Calculate distance between coordinates (shortest path)
  4. QGIS: add coordinates on layer
  5. QGIS: Save raster files to appropriate projection (WS84)
  6. QGIS: Sample from points & extract value from raster file for points
  7. QGIS: Repeat step 6 to get ground elevation, heat and storm water depth for points
  8. QGIS: Format and copy the data to json
